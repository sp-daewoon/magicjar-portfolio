8. 함정·교훈 (사용자 메모리 보강 후보)

  • 병렬 sub-agent 작업 시 cwd 공유 충돌 — 같은 워킹디렉토리에서 4 sub-agent 동시 작업하면 stash 부산물 누적 + branch 전환 race. 다음 사이클부터 worktree 분리 강력 권고 (사용자 메모리 feedback_worktree_cleanup.md 정합).
  • rebase 충돌 자율 해결 — 머지 후 main 변경 시 다음 PR이 conflict. --ours 채택 + force-with-lease 패턴으로 자율 처리 (feedback_pr_rebase_sync.md 정합).
  • stale jar 함정 2회 발생 — docker restart만 사용 시 옛 jar 부팅. scripts/build-and-deploy.sh 정석 경로 강제 의무.
  • destructive 액션 권한 거부 — git reset --hard·git push --force-with-lease가 auto mode에서도 거부됨. 단순 우회(직접 명령) 또는 사용자 명시 승인 필요.
